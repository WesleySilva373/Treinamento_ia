# DevOps Fundamentals: Bridging Development and Operations Culture

## 🚀 Sobre o Projeto
Este repositório reúne o conhecimento sobre a cultura **DevOps**, que surgiu para derrubar o "muro" de comunicação entre os times de desenvolvimento e operação. O foco está na transição do modelo de "software de prateleira" para o **Software as a Service (SaaS)**, que exige entregas contínuas e granulares para o usuário.

## 🧠 Conceitos Chave
*   **Definição de DevOps:** É uma combinação de **filosofia cultural, práticas e ferramentas** que aumentam a capacidade de entrega de serviços em alta velocidade.
*   **Ciclo de Vida:** Um processo contínuo de planejar, codificar, construir, testar, liberar, operar e monitorar.
*   **Cultura Primeiro:** A cultura deve ser estabelecida antes das ferramentas e práticas para garantir o sucesso da organização.

## 🛠️ Gestão de Código Fonte (SCM) com Git
O projeto foca no **Sistema de Controle de Versão Distribuído**, utilizando o **Git**.
*   **Workflow Local:** Os arquivos passam pelos estados de **modificado** (modified), **preparado** (staged) e **confirmado** (committed).
*   **Snapshots:** Diferente de outros sistemas, o Git trabalha com "fotos" do estado do repositório em vez de apenas registrar diferenças (deltas).
*   **Branches e Merges:** Uso de ramificações (branches) para experimentos seguros que, se bem-sucedidos, são integrados à linha principal (master) via **merge**.
*   **Sincronia Remota:** Comandos `git clone` para baixar o repositório, `git push` para enviar mudanças e `git pull` para atualizar o ambiente local.
